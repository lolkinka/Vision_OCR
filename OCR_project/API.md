# 📑 Документация API веб-приложения «Видение» (Vision) v1.0.0

**Vision** — это клиент-серверная система для оптического распознавания (OCR) математических выражений, матриц и систем уравнений на базе локальной нейросети GLM-OCR (Ollama). Приложение поддерживает полноценную сессионную авторизацию пользователей с хранением данных в СУБД PostgreSQL.

---

## 🛠️ Спецификация эндпоинтов (Архитектура REST API)

### 👥 Группа: Авторизация и Навигация (`default`)

#### 1. Отображение страницы входа
* **Метод:** `GET`
* **Путь:** `/login`
* **Описание:** Возвращает HTML-страницу авторизации (`login.html`) с формой ввода данных.
* **Тип ответа:** `text/html`

#### 2. Аутентификация пользователя
* **Метод:** `POST`
* **Путь:** `/login`
* **Описание:** Проверяет учетные данные пользователя. При успешном входе генерирует JWT-токен и записывает его в безопасную куку браузера.
* **Тело запроса (Form Data):**
  * `username` (string, required) — имя пользователя.
  * `password` (string, required) — чистый текст пароля.
* **Тип ответа:** `RedirectResponse` (Перенаправление на `/dashboard` со статус-кодом `303`). При ошибке перерендеривает страницу `/login` с выводом ошибки в контекст.

#### 3. Выход из системы
* **Метод:** `GET`
* **Путь:** `/logout`
* **Описание:** Удаляет сессионную куку `access_token` из браузера пользователя.
* **Тип ответа:** `RedirectResponse` (Перенаправление на `/login` со статус-кодом `303`).

#### 4. Отображение страницы регистрации
* **Метод:** `GET`
* **Путь:** `/register`
* **Описание:** Возвращает HTML-страницу создания нового аккаунта (`register.html`).
* **Тип ответа:** `text/html`

#### 5. Регистрация нового пользователя
* **Метод:** `POST`
* **Путь:** `/register`
* **Описание:** Проверяет уникальность логина, хэширует пароль алгоритмом `bcrypt` с солью и создает запись в PostgreSQL.
* **Тело запроса (Form Data):**
  * `username` (string, required) — желаемый логин.
  * `password` (string, required) — пароль.
* **Тип ответа:** `HTMLResponse` (Возврат на страницу `/login` с зеленой плашкой успешной регистрации).

#### 6. Корневой маршрут
* **Метод:** `GET`
* **Путь:** `/`
* **Описание:** Точка входа в приложение. Перенаправляет на `/login` (если пользователь не авторизован) или на `/dashboard` (если сессия активна).

#### 7. Главный экран (Панель управления)
* **Метод:** `GET`
* **Путь:** `/dashboard`
* **Описание:** Возвращает защищенный рабочий интерфейс сканера формул (`dashboard.html`).
* **Защита:** Требуется валидный JWT-токен в Cookies (`get_current_user`).
* **Тип ответа:** `text/html`

#### 8. Личный кабинет пользователя
* **Метод:** `GET`
* **Путь:** `/profile`
* **Описание:** Отображает профиль текущего пользователя и таблицу истории из 10 его последних успешных конвертаций.
* **Защита:** Требуется валидный JWT-токен в Cookies (`get_current_user`).
* **Тип ответа:** `text/html`

### 🤖 Группа: Нейросетевой процессинг (`Math OCR`)

#### 1. Распознавание математического выражения
* **Метод:** `POST`
* **Путь:** `/predict-math`
* **Описание:** Принимает изображение, передает его в инференс локальной модели `glm-ocr` через Ollama, фильтрует синтаксис от зацикливания и кавычек, логирует операцию в историю БД и возвращает чистый код LaTeX.
* **Защита:** Доступно только авторизованным пользователям.
* **Тело запроса (Multipart/Form-Data):**
  * `file` (binary, required) — файл изображения (форматы: png, jpg, jpeg).
* **Формат успешного ответа (JSON):**
  ```json
  {
    "success": true,
    "formula_latex": "(a+b)^2 = a^2 + 2ab + b^2"
  }
